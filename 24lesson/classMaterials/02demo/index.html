Shared State

Multiple functions can close over the same scope instance and share its bindings. deposit, withdraw, and check are three separate function objects — but they all hold a reference to the same private balance binding. This is the module pattern.

the factory
function makeWallet(initial) {
  let balance = initial;   // private — unreachable from outside

  return {
    deposit:  (amount) => { balance += amount; return balance; },
    withdraw: (amount) => { balance -= amount; return balance; },
    check:    ()       => balance
  };
}

const wallet = makeWallet(100);
// wallet.balance  → undefined  (no direct access)
// wallet.deposit, .withdraw, .check  share the same closed-over scope

what to notice
  • deposit, withdraw, and check are three function objects that all point into the same scope — mutations by one are immediately visible to the others.
  • wallet.balance is undefined. The binding is not a property; it's a lexical variable inside the closed-over scope. No amount of property access can reach it.
  • This is the module pattern — private state, public interface — without a class, without this, without Symbol.
